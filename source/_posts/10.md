主要概念总结：
Entity: 游戏对象的 ID
Component: 数据容器
System: 处理组件的函数
Resource: 全局共享数据
Event: 系统间通信
Query: 查询具有特定组件的实体
Commands: 用于创建/删除实体和组件
Plugin: 组织相关系统和资源的方式


## 实体
Entity (实体) - 游戏对象的唯一ID
Entity 就像是游戏世界中每个对象的"身份证号码"，它只是一个数字ID。 

## 输出两个实体
```rust
use bevy::prelude::*;

fn main() {
    App::new()
        .add_plugins(MinimalPlugins)
        .add_systems(Startup, create_entities)
        .run();
}

fn create_entities(mut commands: Commands) {
    println!("=== 创建两个实体并输出ID ===");
    
    let sprite_entity = commands.spawn((
        Transform::from_xyz(0.0, 0.0, 0.0),
    )).id();
    
    let image_node_entity = commands.spawn((
        Transform::from_xyz(100.0, 0.0, 0.0),
    )).id();
    
    println!("Sprite 实体ID: {:?}", sprite_entity);
    println!("ImageNode 实体ID: {:?}", image_node_entity);
    
}
```

0v0 和 1v0 是 Bevy Entity ID 的简化显示格式，其中：

第一个数字 (0, 1) 是 index：实体的索引编号
v 是分隔符，表示 "version"
第二个数字 (0, 0) 是 generation：实体的代数

## Entity 结构
```rust
pub struct Entity {
    index: u32,     // 索引：0, 1, 2, 3...
    generation: u32 // 代数：用于防止ID重用冲突
}
```
